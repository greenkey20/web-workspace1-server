JSP(Java Server Page)

JSP란, html 형식 내에서 Java 코드를 쓸 수 있는 "Java 언어"
Servlet(Java 코드 내)에서 응답 화면을 구현했던 복잡함을 보다 간단하게 해결 가능한 기술 -> 즉, JSP의 가장 큰 장점은, Servlet에서는 business logic 처리에만 집중 + presentation logic 처리(응답 화면을 만들어내는 view 역할)는 JSP에서 집중할 수 있게끔 분리

JSP 표현법

1. JSP scripting 원소(element)

JSP 페이지에서 Java 코드를 기술할 수 있게 해주는 기능

  1. 선언문: <%! %>
    멤버변수나 메소드를 선언하기 위해 사용
  2. scriptlet(스크립틀릿): <% %>
    JSP에서 Java 코드를 기술하기 위한 제일 기본적인 표현법-
  3. 표현식/출력식: <%= %>
    Java에서 작성한 값(변수 또는 메소드 호출)을 출력하기 위해 사용

2. 지시어/지시자(directive)

JSP page 전체에 영향을 미치는 정보를 기술할 때 쓰임

표현법:
<%@ page/include/taglib 속성="속성 값" ... %>

  1. page 지시어: 현재 JSP 페이지를 처리하는 데에 필요한 각종 속성들을 기술하는 부분
    • language: 사용할 언어 유형 지정
    • contentType: 웹브라우저가 받아볼 페이지의 형식 및 (나의 질문 = 무엇의?)encoding 방식 지정 e.g. utf-8로 encoding하겠다
    • pageEncoding: JSP 파일에 기록된 Java 코드의 encoding 방식 지정 e.g. Java도 utf-8로 encoding 되어있음
    • import: Java의 import 구문과 같은 의미
    • errorPage: 해당 JSP 상에서 오류가 발생했을 경우 보여줄 에러페이지의 경로 지정
