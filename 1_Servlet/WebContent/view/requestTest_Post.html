post 방식으로 요청 후 응답 페이지 받아보기

특징1. post 방식으로 요청하는 건 url의 body 영역에 데이터를 포함시켜서 요청 -> 사용자가 입력한 값/데이터가 url에 노출되지 않음 -> 보안 유지 가능 -> 로그인, 회원 가입 등의 경우 post 방식이 적합함
특징2. body 영역은 전송하는 길이에 제약이 없음(더 정확하게 말하자면, 날아갈 수 있는 패키지의 길이는 제한이 있는 바, 여러 개로 쪼개서 보내고 붙임) -> 게시판 작성 등의 경우 post 방식이 적합함
특징3. 즐겨찾기는 가능하나, 전달되는 데이터가 url에 노출되어있지 않음 -> 기존에 봤던 응답 화면(e.g. my page 등)을 볼 수는 없을 것임
특징4. 최대 요청받는 시간(time out, 기다리는 시간, 제한시간)이 존재함 -> 요새는 연결 속도가 빨라서 이 이슈 거의 발생하지 않음 -> 나의 질문 = 누가(요청자가? 응답자가?) 기다리는 시간인지 간단하게 하신 설명을 제대로 못 들음(14h25)
