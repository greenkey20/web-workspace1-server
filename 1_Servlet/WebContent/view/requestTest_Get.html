get 방식으로 요청 후 응답 페이지 받아보기

특징1. get 방식으로 요청하는 건 url의 header 영역에 데이터를 포함시켜서 요청함 -> 사용자가 입력한 값/데이터가 url에 노출됨 -> 보안에 취약함 -> 로그인, 회원 가입 등의 경우 get 방식은 부적합함
특징2. header 영역은 전송하는 데이터의 길이에 제한이 있음 -> 방대한 데이터를 담았을 경우 초과된 데이터는 절단되어/잘려서 넘어감 -> 게시판 작성 등의 경우 get 방식은 부적합함
특징3/장점. url 데이터가 노출되기 때문에 즐겨찾기/bookmark 기능(url을 저장/즐겨찾기 해두었다가, 필요할 때 클릭하면 해당 그 url 재요청/이동 가능) -> 검색 기능의 경우 get 방식이 가장 적합함
